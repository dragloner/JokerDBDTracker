# JokerDBDTracker Roadmap (до 2.0)

Обновлено: `2026-04-24`
Текущая версия: `v1.4.1`

## Что вошло в `v1.4.1` (патч к `v1.4.0`)

### Чат и комментарии в плеере

- [x] Левый док чата: YouTube chat replay / live chat в отдельной панели слева от видео.
- [x] Нижний док комментариев: YouTube-комментарии в отдельной панели снизу.
- [x] Горячие клавиши переключения доков (C / V), настраиваются в Настройки → Управление.
- [x] Браузер чата полностью останавливается при закрытии панели (navigate to about:blank).
- [x] Фоновое видео заглушено на уровне прототипов HTMLMediaElement до запуска кода YouTube.
- [x] Автоматическая перезагрузка чата при перемотке основного видео (event-based через `seeked`).

### Исправления

- [x] Список таймкодов обновляется сразу после сохранения (без ручного перезапуска).
- [x] Спам-режим звуков больше не складывает один и тот же эффект бесконечно.
- [x] Улучшен HTTP-клиент в сервисе загрузки стримов (Brotli, connection pool, таймауты).

- [x] Поиск по таймкодам.

### Управление

- [x] Настраиваемые горячие клавиши для всех действий плеера через UI настроек.
- [x] Бинды для эффектов 1–15, звуков, скрытия панели.

## Что вошло в `v1.3.1` (патч к `v1.3.0.2`)

### Логирование

- [x] Структурированный формат логов: `[timestamp] [INFO /ERROR] [Source] message`.
- [x] Кольцевой буфер в памяти (200 последних записей) — логи хранятся даже без файла.
- [x] Просмотрщик логов прямо в настройках приложения — обновляется в реальном времени.
- [x] Фильтр «Только ошибки» — скрывает INFO-события, показывает только ERROR.
- [x] Кнопки «Копировать» (все логи в буфер обмена) и «Очистить» (очистить буфер).
- [x] Ошибки выделяются красным цветом в просмотрщике для быстрой диагностики.

### Интерфейс

- [x] Выровнены отступы в панели настроек — везде единый шаг 8/12/16 пикселей.
- [x] Исправлены непоследовательные `Margin` у чекбоксов анимаций и логирования.
- [x] Секции «Режим звука» и «Бинды эффектов» теперь имеют стандартный отступ 16px.

### Звук

- [x] Обновлены имена звуковых файлов: `aura.mp3`, `laugh.mp3`, `psi.mp3`, `donmafia.mp3`.
- [x] Новая настройка «Режим спама» — спам до 8 одновременных звуков вместо stop-and-play.

## Что вошло в `v1.3.0.2` (патч к `v1.3.0.1`)

### Безопасность

- [x] Исправлена потенциальная инъекция в PowerShell-скрипте обновления (санитизация путей).
- [x] Исправлена потенциальная JS-инъекция через VideoId в WebView2 (строгая regex-валидация).
- [x] Усилена защита от path traversal при распаковке ZIP-обновлений.
- [x] DPAPI-шифрование настроек и истории теперь использует entropy (обратная совместимость сохранена).
- [x] Добавлена валидация сетевых сообщений Watch Together (whitelist типов, ограничение позиции и текста).
- [x] Логирование теперь редактирует URL и пути пользователя, добавлена ротация логов (5 МБ).

### Watch Together

- [x] Автоматическая настройка брандмауэра Windows при первом открытии Watch Together.
- [x] Диалог первого запуска с полным объяснением сетевых требований и безопасности.
- [x] Инструкция для ручной настройки если UAC отклонён.
- [x] Согласие пользователя сохраняется в настройках (повторно не спрашивает).

## Что вошло в `v1.3.0.1` (патч к `v1.3.0`)

- [x] Синхронизация звуковых эффектов между участниками.

## Что вошло в `v1.3.0` (по сравнению с `v1.2.3.2`)

### Watch Together (совместный просмотр по LAN / Radmin VPN)

- [x] Добавлена функция Watch Together — совместный просмотр стримов с друзьями.
- [x] Хост создаёт комнату (TCP-сервер), гости подключаются по IP и порту.
- [x] Синхронизация play / pause / seek между хостом и гостями в реальном времени.
- [x] Периодическая автокоррекция позиции (если рассинхрон > 3 сек).
- [x] Синхронизация всех видеоэффектов от хоста к гостям.
- [x] Автоматическое открытие видео у гостей при выборе хостом.
- [x] Определение IP-адресов с приоритетом Radmin VPN (26.x.x.x).
- [x] Полный UI: статус сервера, список подключённых, роль (хост/гость), инструкции.
- [x] Поддержка до 10 одновременных подключений.
- [x] Защита от зависания при подключении (таймаут 5 сек).
- [x] Потокобезопасная запись в сокеты (SemaphoreSlim на каждый поток).
- [x] Корректная работа с локалями (InvariantCulture для чисел в JS).
- [x] Локализация интерфейса (русский / английский).

- [ ] Обновления и восстановление должны работать надёжно
